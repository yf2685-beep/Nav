How the memory works

RetrievalHead
goal DINO vs every past frame → "is it in memory?" (revisit_gate) + a learnable NULL
→
Revisit · seen
fuse current + goal camera pose → head toward the remembered place
Novel · unseen
6-ch early-fusion DINO → visual bearing, then explore
→
DDPM decoder
gate biases attention between revisit & novel → 24-waypoint trajectory

Everything in LingBot stays frozen; only the four heads + decoder are trained. No critic — collision is checked geometrically at eval.

What's strong, what isn't

Works

  • Coarse memory recognition. Cleanly separates seen from unseen goals (AUC 0.924); when a place was never seen, it says so 99.4% of the time.
  • Lossless frozen backbone streamed live, bit-identical to the offline cache.
  • Full closed loop in code — a robot could stream frames to the server and get trajectories back today.

Weak / open

  • Fine localization. Knows whether a goal is remembered, not which exact frame — best real frame within ±5 of the goal is 31% (vs ~5% chance). Adjacent frames look near-identical, so the softmax mass spreads and NULL wins the argmax.
  • Pilot scale — 40–130 trajectories, not a full run.
  • IsaacSim closed-loop — starts headless fine, but scenes_home renders black (broken MDL materials); no textured cluttered_hard USD yet.
