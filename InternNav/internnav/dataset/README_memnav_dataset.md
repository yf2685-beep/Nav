# MemNav Data Loader

`memnav_dataset_lerobot.py` ‚Äî the dataset + collate function for **MemNav**
(implicit-memory, image-goal navigation). One training sample is an
`(episode, goal)` pair drawn from the mp3d revisit episodes generated by
`MemNavData/generate_twoleg.py`. Each sample produces:

- a **NavDP-style action label** (forward path from the current step to the goal),
- a **multi-positive retrieval label** (which history frames co-observe the goal), and
- **pointers + raw images** so the frozen LingBot front-end and dense DINO run GPU-side.

The frozen KV cache is *not* loaded here ‚Äî the loader stays light and emits pointers
(`cache_path`, `rgb_dir`, `cur_step`, `goal_step`) that the policy consumes on-device.

---

## 1. What each episode looks like on disk

Per trajectory directory (`.../<group>/<scene>/<traj>/`):

| File | Purpose |
|---|---|
| `data/chunk-000/episode_000000.parquet` | camera extrinsics / intrinsics (poses) |
| `data/chunk-000/path.ply` | path + obstacle points (kept for signature compat; no critic) |
| `videos/chunk-000/observation.images.rgb/<i>.jpg` | raw frames (window recompute + lazy match reads) |
| `videos/chunk-000/lingbot_cache.npz` | frozen LingBot KV cache + `dino_cls` retrieval keys |
| `meta/gen_meta.json` | per-goal `covis_curve`, `switches`, thresholds |
| `goal_{j+1}.jpg` | rendered goal view for covis goal `j` (a real target view, not a traj frame) |

`lingbot_cache.npz` contains `scale_k/scale_v`, `anchor_k/anchor_v`, and
`dino_cls [T, 1024]`. **Only `dino_cls` is read here** (as the retrieval key space);
the heavy K/V stays on disk and is loaded by the policy via `cache_path`.

When `feature_root` is set, the cache lives in a *separate writable tree* (mirrors the
frame layout) ‚Äî used when frames sit in a read-only squashfs overlay.

---

## 2. Multi-leg structure ‚Üí goals

Each episode is a real 2-/3-leg trajectory. `switches` marks the frame index where
each leg ends. Goals are:

- **Goal A** ‚Äî the first milestone (arrival at end of leg 1). No rendered goal image
  and **no `covis_curve`**. Under the unified revisit rule (¬ß4) A has no earlier pass,
  so it is **always novel** ‚Äî a clean negative/novel training sample. Uses the last
  frame of leg 1 as its goal image (`has_covis=False`).
- **Goals B, C, ‚Ä¶** ‚Äî one per leg switch, each with a rendered `goal_{j+1}.jpg` and a
  `covis_curve`: occlusion-aware co-visibility of the goal view against every prior
  history frame `[0..k]` (index `i` aligns with global frame `i`). `has_covis=True`.

Goals are enumerated once at construction in `_parse_meta` ‚Üí flat `self.samples` list.

---

## 3. Sampling the current step `k`

`k` is **not** stored; it is sampled per `__getitem__` inside the goal's own leg:

```
k ~ Uniform[k_lo .. k_hi]
  k_lo = max(leg_start, anchor_margin + exclude_recent)   # covis goals
  k_lo = anchor_margin + exclude_recent                    # goal A
  k_hi = goal_step - goal_slack                            # >= goal_slack forward frames remain
```

The `+ exclude_recent` floor guarantees the candidate region `E(k) = [anchor_margin ..
k ‚àí exclude_recent]` (¬ß4) is non-empty.

`anchor_margin = num_scale + window_size - 1` (default `8 + 32 - 1 = 39`). This floor
guarantees the memory partitions into three **disjoint** contiguous regions used by the
LingBot window-forward:

```
scale   frames [0 .. 7]        full K/V
history frames [8 .. k-W]      6 special tokens only
window  frames [k-W+1 .. k]    recomputed live from raw RGB (in the policy)
```

Goals whose leg is too short to satisfy `k_hi >= k_lo` are dropped in `_parse_meta`.

---

## 4. The retrieval label (`_build_label`) ‚Äî unified `E(k)` rule

**One rule for A, B, and C, identical at train and inference** (needs only `k`, no leg
structure). A history frame is a **revisit candidate** only if it is at least
`exclude_recent` frames behind the current step:

```
E(k) = [ anchor_margin .. k ‚àí exclude_recent ]        # exclude_recent = 83
```

Frames nearer than `exclude_recent` to `k` are the **current approach** ‚Äî the robot sees
the goal *because it is walking toward it* ‚Äî and are routed to the novel / in-FoV branch,
**not** revisit. `exclude_recent` is the covis-onset horizon (p90 of the covis‚â•0.1 onset
distance): any approach frame lies within `onset` of the goal, and `k ‚â§ goal`, so within
`onset` of `k` ‚Üí excluded. Genuine loop closures sit ~hundreds of frames back (measured
median gap **321**), so `t=83` removes only **0.4%** of trainable `k`.

Labels over `E(k)`:

| goal | positive | negative |
|---|---|---|
| **covis B/C** | `covis_i ‚â• covis_pos_hi (0.5)`, `i ‚àà E(k)` | `covis_i ‚â§ covis_pos_lo (0.1)`, `i ‚àà E(k)` |
| **goal A** | *(none ‚Äî no earlier pass)* | all of `E(k)` |

Own-leg / undefined-covis frames are treated as covis 0 (they sit `> t` back, beyond the
onset ‚Üí genuinely don't see the goal). `null_pos` (= **novel**) is **dynamic**: True iff
`E(k)` contains no positive. Consequences:

- **Goal A is always novel** (its only co-observers are recent approach frames, excluded).
- A covis B/C sampled at a `k` where the old match hasn't yet cleared `t` is *novel for
  that k* ‚Äî correct ("I basically just saw it").

**Consumed by two decoupled losses** (not one softmax ‚Äî a null-in-softmax collapses to
always-null). `cand_mask` (= `E(k)`) is emitted alongside `pos_mask`/`neg_mask`:

- **ranking** ‚Äî multi-positive InfoNCE over `pos` vs `pos‚à™neg`, revisit rows only.
- **gate** ‚Äî BCE on `sigmoid(a¬∑max_i cos_i + b)` vs `is_revisit`, over the max cosine
  across `E(k)` (probe: absolute top-1 cosine separates revisit/novel at AUC‚âà0.91).

## 5. The action label (`_build_actions`)

Reuses the NavDP action pipeline (`process_actions`) on the forward segment
`extrinsics[k .. goal_step]` (index 0 = current frame k):

- `action_indexes = clip(arange(predict_size+1) * pred_digit, 0, L-2)` ‚Äî with
  `predict_size=24`, `pred_digit=4` this reaches up to 96 frames ahead and **saturates
  at the goal** when the segment is shorter (arrive-and-stop ‚Üí zero deltas). No `k ‚â§ T‚àí24`
  constraint.
- `pred_actions [24, 3]` = `(pred_xyt[1:] - pred_xyt[:-1]) * 4.0` (local x, y, Œ∏ deltas).
- `goal_rel_pose [3]` = the **true** full-path endpoint (not the horizon-truncated
  waypoint) ‚Äî GT for the revisit aux-pose head.

`random_digit=True` samples `pred_digit ‚àà [2,8)` per item; default is fixed `pred_digit=4`.

## 6. `__getitem__` output (per sample)

| Key | Shape | Notes |
|---|---|---|
| `mem_cls` | `[k+1, 1024]` | `dino_cls[:k+1]` retrieval keys |
| `cand_mask` | `[k+1]` bool | revisit candidate region `E(k) = [amargin..k‚àít]` |
| `pos_mask` / `neg_mask` | `[k+1]` bool | multi-positive label within `E(k)` |
| `null_pos` | scalar bool | novel (True) vs revisit (False) ‚Äî **dynamic per k** |
| `is_revisit` | scalar float | `1 - null_pos` |
| `pred_actions` | `[24, 3]` | action deltas |
| `goal_rel_pose` | `[3]` | aux-pose GT |
| `goal_image` | `[3, H, W]` | LingBot-preprocessed real goal view |
| `window_images` | `[W, 3, H, W]` | current local window `[k-W+1 .. k]` |
| `cache_path`, `rgb_dir`, `cur_step`, `goal_step` | pointers | policy loads KV cache + lazy match frames GPU-side |

Images use LingBot's exact preprocessing (`load_and_preprocess_images`, square-pad to
`image_size=518`) so they match what the GCT window-forward + dense DINO expect.
Goal CLS is **not** cached (real goal images) ‚Äî the policy computes it from `goal_image`
via the frozen context-free DINO trunk.

---

## 7. `memnav_collate_fn`

- Stacks fixed-shape tensors (`pred_actions`, `goal_rel_pose`, `goal_image`,
  `window_images`, `null_pos`, `is_revisit`).
- **Pads** variable-length `mem_cls` / `cand_mask` / `pos_mask` / `neg_mask` to `Lmax`;
  padded frames are `False` in `mem_mask`/`cand_mask` (and pos/neg) ‚Üí they drop out of
  both attention scores and the retrieval loss.
- Keeps pointers (`cache_paths`, `rgb_dirs`, `cur_steps`, `goal_steps`) as length-B lists.

Output keys are prefixed `batch_*` and consumed by `MemNavTrainer.compute_loss`:
**decoupled** ranking InfoNCE (`pos` vs `pos‚à™neg`, revisit rows) + gate BCE
(`sigmoid(a¬∑max_cos+b)` vs `is_revisit`, all rows) + aux pose on revisit rows only.

---

## 8. Key constructor knobs

| Param | Default | Meaning |
|---|---|---|
| `predict_size` | 24 | action waypoints |
| `pred_digit` / `random_digit` | 4 / False | frame stride per waypoint |
| `num_scale` / `window_size` | 8 / 32 | memory partition geometry (must match precompute) |
| `anchor_margin` | `num_scale+window-1`=39 | earliest labelable frame (candidate-region floor) |
| `exclude_recent` | 83 | revisit candidate horizon: frames within `t` of `k` are approach‚Üínovel (p90 covis-onset) |
| `covis_pos_hi` / `covis_pos_lo` | 0.5 / 0.1 | covis pos/neg thresholds (meta overrides per-episode) |
| `glimpse_pos` / `glimpse_neg` | 14 / 83 | *(legacy ‚Äî superseded by `exclude_recent`; unused in the unified rule)* |
| `goal_slack` | 4 | min forward frames `k ‚Üí goal_step` |
| `add_goalA` | True | include the goal-A milestone samples |
| `feature_root` | None | separate writable cache tree (read-only frame overlays) |
| `scene_data_scale` / `trajectory_data_scale` | 1.0 | subsample scenes / trajectories |

Run `python -m internnav.dataset.memnav_dataset_lerobot --root_dirs <path> --n 20`
for a standalone sanity check (revisit fraction, mask/mem alignment, collate shapes).
